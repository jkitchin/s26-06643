# Software Forge Honors - Ranks

Four ranks representing progressive mastery in scientific software development. Each rank builds on the previous, culminating in Master Craftsperson status.

---

## Rank Progression

```
Code Apprentice → Software Crafter → Software Engineer → Master Craftsperson
    (Week 4)         (Week 8)          (Week 12)            (Week 15)
```

---

## Rank 1: Code Apprentice

*"Every master was once a beginner."*

**Symbol:** 🔧 (Wrench)

**Timeline:** Achievable by Week 4

### Requirements

| Requirement | Details |
|-------------|---------|
| **Badges** | 1 badge (typically Toolsmith) |
| **Quizzes** | Passing average on lectures 00-02 |
| **Capstone** | Reflection essay |

### Capstone Activity

**Write a 1-page reflection:**
*"How AI-Assisted Development Changed My Coding Workflow"*

Prompts to address:
- What surprised you about working with AI assistants?
- How has your approach to problem-solving changed?
- What do you still prefer to do manually vs. with AI help?

### Recognition
- Digital badge for portfolio
- Listed in course recognition page

## Rank 2: Software Crafter

*"Craftsmanship comes from practice and attention to detail."*

**Symbol:** 📦 (Package)

**Timeline:** Achievable by Week 8

### Requirements

| Requirement | Details |
|-------------|---------|
| **Badges** | 2 badges |
| **Midterm** | Score 70% or higher |
| **Capstone** | Code review OR tutorial presentation |

### Capstone Activity (choose one)

#### Option A: Code Review Session
Participate in a structured code review:
- Review a peer's code (give constructive feedback)
- Have your code reviewed (receive and respond to feedback)
- Document what you learned from both perspectives

#### Option B: Tutorial Presentation
Create and present a 10-minute tutorial on a course concept:
- Topics: Git workflows, pytest fixtures, CI/CD setup, etc.
- Include live coding or demonstration
- Prepare practice exercises for audience

### Recognition
- Digital badge for portfolio
- Peer review/teaching opportunities
- Study group leadership role

## Rank 3: Software Engineer

*"Engineers build things that work reliably."*

**Symbol:** ✅ (Check Mark)

**Timeline:** Achievable by Week 12

### Requirements

| Requirement | Details |
|-------------|---------|
| **Badges** | 4 badges |
| **Performance** | Maintained 75%+ quiz average |
| **Field Experience** | Complete one real-world experience |
| **Capstone** | Full software project |

### Field Experience Options (choose one)

| Experience | Description | Documentation |
|------------|-------------|---------------|
| **Open Source Contribution** | Contribute to an open source project | Link to PR/issue and reflection |
| **Industry Interview** | Interview a professional software engineer | 1-page summary of insights |
| **Research Tool Development** | Build a tool for a research group | Tool with documentation |
| **Professional Event** | Attend a software development meetup or conference | Summary and key takeaways |

### Capstone Activity

**Complete a Full Software Project:**
- Build a useful tool or library
- Include comprehensive tests (80%+ coverage)
- Set up CI/CD pipeline
- Write clear documentation

**Deliverables:**
1. GitHub repository with clean history
2. Published documentation
3. 2-page project summary

### Recognition
- Digital badge for portfolio
- Potential TA or peer mentor role
- Project may be featured in course materials

## Rank 4: Master Craftsperson

*"Masters not only create—they teach and inspire others."*

**Symbol:** 🏆 (Trophy)

**Timeline:** Achievable by Week 15

### Requirements

| Requirement | Details |
|-------------|---------|
| **Badges** | All 5 badges |
| **Final Exam** | Score 70% or higher |
| **Project** | Complete semester project with distinction |
| **Capstone** | Portfolio presentation |

### Capstone Activity

**Portfolio Presentation:**

Present your semester's work (15-20 minutes) demonstrating:

1. **Technical Growth**
   - Show progression from early to late semester code
   - Highlight improvements in quality, testing, documentation

2. **Project Showcase**
   - Demo your main project
   - Explain architecture decisions
   - Show test coverage and CI/CD pipeline

3. **Collaboration Evidence**
   - Pull requests given and received
   - Code reviews conducted
   - Open source contributions

4. **Documentation Quality**
   - Published docs or tutorials
   - Clear README files
   - API documentation

5. **Reflection**
   - What you'd do differently
   - Skills you want to develop next
   - How you'll apply these skills

### Recognition
- Digital badge for portfolio
- Certificate of Distinction
- Project archived in course materials
- Strong recommendation letter support
- Invitation to mentor future students

## Notes for Success

### Build in Public
Put your work on GitHub from day one. Your commit history tells the story of your growth.

### Start Small with Open Source
Your first contribution doesn't need to be code—documentation improvements, typo fixes, and issue reports all count.

### Document As You Go
Don't save documentation for the end. Write docstrings as you code, update README as you build.

### Ask for Reviews
Don't wait for formal review assignments. Ask peers to review your code—and offer to review theirs.

### Embrace Iteration
Your first draft won't be perfect. The craft is in the refining.
